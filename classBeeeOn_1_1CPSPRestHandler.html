Detailed Description

is intended to server CPSP pages via the REST framework. All webpages must be compiled into classes and registered with the CPSPRestHandler instance. This can be done via the dependency injection. Each registered page must be annotated via the BEEEON_OBJECT_* macros.

The REST mapping to actions is done via the class name of the CPSP page. If a class MyNS::IndexHandler is generated for a page, then the mapping is done as:

api.call.cpsp.MyNS::IndexHandler.uri = /myns/index

No arguments are possible. To support arguments, you must use URI wildcards like /myns/index/* and parse the URI manually by the page.
