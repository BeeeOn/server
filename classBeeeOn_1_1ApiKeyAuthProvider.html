Detailed Description

ApiKeyAuthProvider provides authorization of clients identified by a fixed API key. There is a file containing mapping of API keys to users (keyFile) that is used for authorization.

The keyFile is loaded via Poco::Util::PropertyFileConfiguration. An example of keyFile contents: 

ahqueeshiech4nasevaiY4Vi6shai9eiYeichien = joe.d.nosp@m.oe@e.nosp@m.xampl.nosp@m.e.or.nosp@m.g
Koofae5xohsh5ilohxoo5ahw3aiy4LuofeeBuef0 = john..nosp@m.smit.nosp@m.h@exa.nosp@m.mple.nosp@m..org
xahneilee5ewiozech5pheiBoaJathei6phai8ie = micha.nosp@m.el.j.nosp@m.ackso.nosp@m.n@ex.nosp@m.ample.nosp@m..org
tah2eirieghohwie2Ileej1ahshoraiRohsephah = fredd.nosp@m.y.me.nosp@m.rcury.nosp@m.@exa.nosp@m.mple..nosp@m.org
tah2eirieghohwie2Ileej1ahshoraiRohsephah.first_name = Freddy
tah2eirieghohwie2Ileej1ahshoraiRohsephah.last_name = Mercury
tah2eirieghohwie2Ileej1ahshoraiRohsephah.locale = en_GB

It is possible to specify only the mapping as <KEY> = <EMAIL>. However, first_name, last_name and locale can be optionally given for each <KEY>.

The keyFile is reloaded when changed. The reload is performed on demand but only once per the reloadPeriod (unless it is zero).
