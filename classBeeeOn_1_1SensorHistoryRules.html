Detailed Description

SensorHistoryRules implements check of parameters used when asking for sensor history data. In this way, it is possible to restrict requests to a supported range of request or to forbid requests which are too computational intensive.

◆ parseSpan()

SensorHistoryRules::Span SensorHistoryRules::parseSpan ( const std::string &  input)
static

Parse the given span specification.

It accepts inputs like: 1 s, 1 m, 5 s..1 m, 5 s..1 m+, 5 m..max.

The '+' suffix is used to determine inclusive interval. Consider that 1 s..1 m is equivalent to 1000000 us..60000000 us. But 1 s..1 m+ is equivalent to 1000000 us..60000001 us.

The syntax rules:

  • NUMBER UNIT
  • max
  • NUMBER UNIT PLUS?..NUMBER UNIT PLUS?
  • NUMBER UNIT PLUS?..max

The NUMBER is treated as [0-9]+.

The UNIT is treated as {us|ms|s|m|h|d|} (empty is possble).

The PLUS is treated as +
