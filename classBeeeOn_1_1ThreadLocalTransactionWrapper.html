Detailed Description

A thread-local transaction is used in the most common transactional strategy. Every thread has no or a single transaction active at anytime. This means that we can access such transaction anytime in the code flow where we have access to the appropriate TransactionManager instance.

The ThreadLocalTransactionWrapper has the pointer to the ThreadLocal instance holding it. When the ThreadLocalTransactionWrapper is destroyed (by delete), the reference in the ThreadLocal is cleared.
