Detailed Description

Non-verified identity representation. Before a user is registered in the system, it is possible to represent it as an identity. An identity can be verified or non-verified. An identity is verified when a user logs into the system by using the identity's email and some 3rd party service (Google, Facebook, ...). This class represents any identity in the system and has a globally unique e-mail.
