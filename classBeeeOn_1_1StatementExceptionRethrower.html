Detailed Description

StatementExceptionRethrower allows to reformat exception messages with respect to the associated statement and rethrow such exceptions in new better form.

The database exceptions can be of an implementation-specific type. Such exceptions are usually not linked into the main application (and loaded dynamically). Thus, we are not able to hard-wire their processing into the code. It is possible to register specific instances of StatementExceptionRethrower during runtime to handle exceptions from external libraries.
